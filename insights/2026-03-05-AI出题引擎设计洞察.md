# 产品洞察：AI 出题引擎的设计方向

> 基于 06 号化学老师访谈 + 跨访谈综合分析，2026-03-05

---

## 核心洞察

### 老师不需要"自动出题机"，需要"出题合作者"

从 6 次访谈中，教师使用 AI 出题的真实模式浮现：

```
❌ 老师期望的（我们以为的）：
   "帮我出 10 道极坐标题" → AI 一次性输出 10 道完整好题

✅ 老师实际的工作流：
   对话 1: "极坐标有哪些真实应用场景？"
   对话 2: "GPS 定位这个方向不错，围绕这个出题"
   对话 3: "第 1 问太简单了，加个坐标转换"
   对话 4: "第 3 问超纲了，删掉微积分部分"
   对话 5: "帮我生成一个极坐标图"
   对话 6: "最终版确认，导出为试卷格式"
```

**产品应该是对话式逐步打磨，不是一键生成。**

## 出题引擎架构建议

### 1. 三层出题策略

```
┌─────────────────────────────────────┐
│          场景选择层                   │
│  课堂练习 | 章节测验 | 期末考试      │
└──────────────┬──────────────────────┘
               │
┌──────────────▼──────────────────────┐
│          认知层级框架                 │
│  布鲁姆分类法自动分配难度梯度         │
│                                     │
│  记忆 → 理解 → 应用 → 分析 → 评估   │
│  第1问   第2问   第3问   第4问       │
└──────────────┬──────────────────────┘
               │
┌──────────────▼──────────────────────┐
│          背景生成层                   │
│  真实场景库 + 知识点映射              │
│  Supermemory 检索 + Perplexity 搜索  │
└─────────────────────────────────────┘
```

### 2. 对话式打磨流程

```
Step 1: 场景 + 知识点确认
  AI: "您想考哪个知识点？用于什么场景？"
  老师: "氧化还原反应，章节测验"

Step 2: 背景提案
  AI: "我找到几个真实背景：
       1. 铁锈蚀与防护
       2. 食品抗氧化剂
       3. 电池工作原理
       您倾向哪个？"
  老师: "食品抗氧化剂"

Step 3: 题目结构设计
  AI: "基于布鲁姆分类法建议：
       Q1（理解）: 写出维C的氧化反应方程式
       Q2（应用）: 计算某食品中维C含量
       Q3（分析）: 分析图表中保鲜效果与维C浓度关系
       Q4（评估）: 比较两种抗氧化方案的优劣"
  老师: "Q3 太难了，改成简单的数据读取"

Step 4: 精修 + 格式化输出
  AI: 生成完整试题（含方程式渲染、图表）
  老师: 确认 → 导出
```

### 3. 学科专项能力

| 学科 | 特殊需求 | 技术方案 |
|------|---------|---------|
| 化学 | 方程式、分子结构 | LaTeX + mhchem 宏包 |
| 数学 | 公式、坐标图 | LaTeX + Desmos API |
| 物理 | 力学图、电路图 | SVG 生成 |
| 生物 | 细胞图、流程图 | 图片生成 AI |
| 通用 | 数据表、柱状图 | Chart.js / D3 |

## 跨访谈验证（6 次访谈汇总）

### AI 出题痛点频率

| 痛点 | 提及次数 | 涉及老师 |
|------|---------|---------|
| 出题质量差/太基础 | 4/6 | 01 环科、02 化学、05 数学、06 化学 |
| 非 AP/非标课程题库空白 | 3/6 | 02 化学、05 数学、06 化学 |
| 无法处理公式/图表 | 3/6 | 05 数学、06 化学、02 化学 |
| 无难度梯度 | 2/6 | 05 数学、06 化学 |
| AI 只能当头脑风暴用 | 2/6 | 01 环科、06 化学 |

### 老师对 AI 的使用模式谱系

```
完全不用 ←───────────────────────→ 深度依赖
   │                                    │
   03语文                    06化学(创意)
                02化学(学案)
                   01环科(出题)
                      05数学(出题)
```

所有老师都落在中间偏左的位置 —— 用 AI 辅助，但不信任 AI 的成品质量。

## MVP 建议

### Phase 1: 对话式出题（最小可行版）

```
输入：学科 + 知识点 + 场景类型
输出：3 个背景方案供选择
  → 老师选择后生成题目框架
  → 支持逐问修改
  → 最终输出结构化试题
```

不需要完美的图表/方程式渲染（Phase 2 再做），先把**对话式打磨流程**跑通。

### Phase 2: 学科渲染

- LaTeX 公式渲染
- 化学方程式（mhchem）
- 简单图表（数据表、折线图）

### Phase 3: 知识图谱 + 教育学框架

- 布鲁姆分类法自动难度分配
- AP/IB 课程知识图谱
- 跨知识点综合出题
